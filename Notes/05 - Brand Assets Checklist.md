# 🎨 Brand Assets Checklist

What's needed for the portal frontend. Tick off as collected.

---

## Identity

- [ ] **Logo** — SVG (preferred), with PNG fallback at 1x / 2x / 3x densities
- [ ] **Wordmark** — separate from logo if it exists (e.g. "Fast Action Claims" lettering only)
- [ ] **App icon** — square, 1024×1024, no transparency (for App Store + Play Store later)
- [ ] **Favicon** — ICO or SVG for web

## Colour

- [ ] **Primary brand colour** — hex
- [ ] **Secondary brand colour** — hex
- [ ] **Accent / signal colour** — hex (used for the "What We Need From You" badge and progress tracker)
- [ ] **Neutrals** — full grayscale ramp from white to near-black
- [ ] **Status colours** — success, warning, error, info (each with light + dark mode variants)
- [ ] **Dark mode palette** — confirm whether dark mode reuses brand colours or has its own ramp

## Typography

- [ ] **Display face** — for headings, prominent figures, hero text
- [ ] **Body face** — for paragraph text, form labels, UI chrome
- [ ] **Licence / source** — Google Fonts, Adobe Fonts, custom-licensed?
- [ ] **Fallback stack** — for systems where the primary face fails to load
- [ ] **Numerals** — tabular figures available? Important for financial summary screen.

## Voice & Tone

- [ ] **Tone of voice guide** — existing document if there is one, else build from scratch alongside frontend brief copy
- [ ] **Existing letterhead** — reference for visual continuity with print correspondence (firm uses City Point, Stretford address per the recent change)
- [ ] **Sample client comms** — emails or letters that read well, for tone reference

## Reference Material

- [ ] **Existing marketing site** — www.rowanrose.co.uk — what works visually, what doesn't?
- [ ] **Competitor portals seen** — what to learn from / avoid?

---

## If Assets Are Not Ready Before Prototyping

Prototypes use:
- Placeholder wordmark in a chosen-during-exploration typeface
- A typographic palette decided per aesthetic direction (each prototype picks its own — that's the whole point of Phase A)
- Status colours from each direction's own palette

Real assets get swapped in during Phase B once direction is locked.
