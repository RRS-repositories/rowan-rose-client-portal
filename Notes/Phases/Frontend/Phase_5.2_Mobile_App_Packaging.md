# Phase 5.2 — Mobile app packaging (PWA + Capacitor Android)

<context>
The portal frontend (`frontend/app`, React 18 + Vite 6) was a responsive web app with
`vite-plugin-pwa` installed but effectively unused (SVG-only icon, no iOS tags). Brad wanted to
explore the app full-screen on his own iPhone and Android phone, looking exactly like the web app,
while developing on Windows 11. Still mock-only; no backend.
</context>

## Goal

The same web app installs and runs full-screen on a phone: as a PWA on both iPhone and Android
today, and as a real native Android app (Capacitor APK). iOS native + store publishing deferred
(Windows can't build iOS).

## Tasks

1. PWA polish: raster icons from `favicon.svg`, manifest icons (192/512/maskable), iOS
   `apple-touch-icon` + web-app meta tags in `index.html`.
2. Capacitor 8 wrap: `capacitor.config.ts`, native splash-hide on first paint, Android hardware
   back-button → router, native icons/splash via `@capacitor/assets`, `android/` scaffolded.
3. Keep the web PWA intact while ensuring native ships **no service worker**.
4. Document run/install for both phones (`frontend/app/MOBILE.md`).

## Out of scope

Native iOS build (needs a Mac/cloud-Mac), App Store / Play Store submission (signed AAB, paid dev
accounts), device-native plugins (camera, push) — all deferred. Still no real backend; mocks only.

## Build notes — what actually happened

- **PWA**: icons generated by `frontend/app/scripts/generate-icons.mjs` (full-bleed navy + "R"),
  NOT `@vite-pwa/assets-generator` — that tool bundles sharp 0.32.6 whose Windows binary fails to
  load (`ERR_DLOPEN_FAILED`). Pinned `sharp@0.33.5` as a direct devDep to fix it; removed the
  unused generator. Manifest gained 192/512/maskable PNG entries; `index.html` gained
  `apple-touch-icon` + `apple-mobile-web-app-*` tags + `viewport-fit=cover`.
- **iOS status bar**: used `apple-mobile-web-app-status-bar-style: default` (NOT the plan's
  `black-translucent`) — the sticky MobileHeader has no safe-area-inset padding, so translucent
  would let the iOS clock overlap the header.
- **Capacitor 8.4**: appId `uk.co.rowanrose.portal`, appName `Rowan Rose`. `android/` and
  `ios` package installed; only **android** platform scaffolded (`cap add android`). iOS deliberately
  not added (Windows).
- **Service-worker conflict**: `VitePWA(...)` in `vite.config.ts` is now gated on
  `process.env.CAP_BUILD === "1"`. Web build (`npm run build`) keeps the PWA; native build
  (`npm run mobile:build`, sets `CAP_BUILD=1`) ships no SW — avoids stale-cache white screens in
  the WebView. Verified: native `dist/` has no `sw.js`/`workbox`.
- **Routing**: kept `BrowserRouter` (Capacitor serves a local origin with index fallback). Added an
  Android hardware back-button listener in `AppShell.tsx` (guarded by `Capacitor.isNativePlatform()`,
  dynamic imports so the web bundle is untouched).
- **Files**: new `capacitor.config.ts`, `scripts/generate-icons.mjs`, `MOBILE.md`, `resources/`
  (native source art), `android/` (committed; build output gitignored). Edited `vite.config.ts`,
  `index.html`, `package.json` (deps + `mobile:build`/`cap:*` scripts), `main.tsx`, `AppShell.tsx`.
- **Toolchain gap**: machine has Node 22 but Java 8 + no Android SDK. Building the APK needs a
  one-time Android Studio + JDK 21 install (steps in `MOBILE.md`). Scaffolding/sync did NOT need it.

## Verification

- `npm run build` → web build emits `sw.js` + `manifest.webmanifest` with all icons (PWA intact).
- `CAP_BUILD=1 npm run build` / `npm run mobile:build` → no `sw.js`/`workbox` in `dist/`.
- `npm run typecheck` clean. `cap add android` + `cap sync android` succeed; 136 Android
  icon/splash assets generated.
- PWA install: `npx vite preview --host`, open `http://<LAN-IP>:4173` on phone → Add to Home
  Screen (iOS Safari / Android Chrome) → launches standalone with the Rowan Rose icon.
- Native Android (after toolchain install): `npm run cap:android` runs on device/emulator; debug
  APK at `android/app/build/outputs/apk/debug/app-debug.apk`.
- Full run/install guide: `frontend/app/MOBILE.md`.
